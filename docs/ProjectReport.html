Repository Purpose

This repo validates a realistic e-commerce flow from login through order confirmation. It uses a controlled mock storefront at https://mock-store.local, fulfilled by Playwright request routing.

The mock store avoids public Amazon bot blocking while preserving the structure of a real shopping experience: header navigation, product search, cart, checkout, payment, and confirmation.

Architecture

Feature Layer Gherkin files describe behavior using business-readable scenarios and tags such as @smoke, @checkout, and @api.
Step Layer Reqnroll step definitions map BDD language to reusable C# actions and assertions.
Page Object Layer Page classes encapsulate selectors and UI behavior for login, search, product, cart, checkout, payment, and confirmation pages.
Support Layer Hooks create Playwright browser contexts, API contexts, tracing, screenshots, videos, and runtime configuration.
Mock Store The mock storefront is generated in C# and routed through Playwright for deterministic browser tests.
CI/CD Layer GitHub Actions runs cross-browser tests and uploads TRX reports plus Playwright failure evidence.
